Задача состоит в создании двусторонней связи между родительским и дочерним процессами с использованием одного неименованного канала и циклической организации обмена данных с помощью семафоров из системы Unix.

Идея реализации следующая:
1. Создается неименованный канал для обмена данными между процессами.
2. Инициализируются семафоры - один для писателя и один для читателя (изначально семафор писателя имеет значение 1, а читателя - 0).
3. Повторяется следующая последовательность действий 10 раз:
   - Родительский процесс ожидает открытия семафора писателя, а дочерний процесс ожидает открытия семафора читателя.
   - Родительский процесс уменьшает значение семафора писателя, записывает случайно сгенерированное число в канал, а затем увеличивает значение семафора читателя.
   - Дочерний процесс, после увеличения семафора читателя родительским процессом, начинает свою работу. Он считывает число из канала, выводит его в консоль, а затем увеличивает значение семафора писателя и завершает свою работу.

Оригинальный исходный код находится в файле main.c.
